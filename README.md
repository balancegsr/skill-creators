# Skill Creator 系列

> **MethodCraft** series — 把经过验证的方法论编译为可执行的 AI Skill

一组 Skill Creator（生成器），根据你的项目需求生成定制化的项目级 Skill。

每个 Creator 运行一次，通过 2-3 轮对话了解你的需求后，生成一个完整的 SKILL.md（+ references/）。生成物安装后，能在你的实际项目中持续引导从启动到结项的全过程。

## 核心特性：自带 Memory 系统

所有生成的 Skill 都内置文件级 memory 机制——这是这组 Creator 的核心差异化价值：

**你只管做事，Skill 替你记住一切。**

- **自动保存**：关键产出物（笔记、决策、方案文档）在触发点自动生成并保存，你只需专注当前任务
- **跨对话恢复**：每次开新对话，Skill 自动读取状态文件、声明当前进度——不需要你重复交代"上次到哪了"
- **结构化组织**：文件按模块/步骤分文件夹存放，项目再大也保持清晰可导航
- **智能读取优先级**：长期项目中文件会越来越多，Skill 自动调整哪些文件优先加载到上下文，不占用宝贵的上下文窗口
- **严禁删除**：所有"归档"行为都是读取优先级的调整，你的原始文件始终保留

这套机制基于纯文件读写实现，不依赖任何平台的记忆 API。换句话说，只要 agent 能读写文件，memory 系统就能工作。

## 家族成员

| Creator | 用途 | 生成物 | 状态 |
|---|---|---|---|
| [learning](./learning/) | 系统学习一个领域/技能 | 学习项目引导 Skill（规划→学习→复盘→结项） | ✅ v1.3.0 |

三个 Creator 覆盖"学→设计→做"的完整链路，共享统一的交互框架和文件管理机制，但在方法论骨架上各自独立——学习用四模式切换，设计用有向五步 + 回退，构建用里程碑驱动 + 四子流可回退执行图。

## 工作原理

```
你的项目需求 → Creator（2-3 轮对话）→ 定制 SKILL.md → 持续引导你的项目
```

Creator 是一次性的生成工具。它的产物——项目级 Skill——才是长期陪你的东西。

不同的输入会产生结构和内容有实质性差异的 Skill。每个 Creator 都提供轻量/完整两种模式，适配不同规模的项目。

## 共享设计

三个 Creator 共享同一套交互框架和设计原则：

- **结构化对话收集**：必答变量 3-5 个，可选变量被动捕获，不像填表
- **预览-迭代-交付**：先给你看生成物预览，满意再交付，随时可提修改
- **两种模式**：轻量模式（短期聚焦）和完整模式（长期系统），覆盖范围不同但引导质量一致
- **文件即 memory**：生成的 Skill 通过结构化文件读写实现跨对话状态保持
- **跨平台路径探测**：安装路径自动适配当前平台的 skill 目录规范

## 兼容性

主要为 [OpenClaw](https://github.com/nicepkg/OpenClaw) 设计，同时兼容能操作本地文件系统的 AI Agent 产品（Claude Code、WorkBuddy、CodeBuddy、Antigravity 等）。所有指令使用描述性语言，不依赖特定平台 API。
